1. 挂载 卸载
 1）管理存储设备的第一步是把 设备连接到文件系统树中，挂载。
     /etc/fstab的文件可以列出系统启动时要挂载的设备

     设备        on 挂载点    
     /dev/disk1s1 on /      (apfs, local, journaled)
     设备/dev/disk1s1在
     devfs on /dev (devfs, local, nobrowse)
     /dev/disk1s4 on /private/var/vm (apfs, local, noexec, journaled, noatime, nobrowse)
     map -hosts on /net (autofs, nosuid, automounted, nobrowse)
     map auto_home on /home (autofs, automounted, nobrowse)

     umount /dev/hdc

  2）一个挂载点就是文件系统树中的一个目录
     Demo：1.创建一个目录  mkdir /mnt/cdrom
           2.mount -t iso9660 /dev/hdc /mnt/cdrom
           3.转换工作目录
           4.umount /mnt/cdrom
               卸载设备需要把所有剩余的数据写入这个设备，所以设备可以安全地移除
               如果没有卸载就移除设备，就有可能没有把注定要发送到设备中的数据输送完毕，这些数据
               可能包含重要的目录更新信息，这将导致文件系统损坏 

2.创建新的文件系统
  fdisk 程序允许我们直接在底层与类似磁盘的设备进行交互
  可用tail -f /var/log/message实时监控获得设备名称
  sudo umount /dev/sdb1
  sudo fdisk /dev/sdb
  
  启动程序后，
  Commad （m for help）
  输入m
  显示 a toggle1.....
  
  输入p，打印出这个设备的分区表
   ...
   ...
   Device Boot     Start       End      Blocks       Id        System
   /dev/sdb1        2         1008      15608+       b         w95 FAT32
   
  更改System分区，
    1. 找到被用来标识一个Linux分区的ID号码 
    2. 输出l ，查看可用的分区类型列表  
    3. 输入t，再输入新的ID号
    4. 输入w，把修改过的分区写入此设备 ，q退出不修改
 
  mkfs 创建各种格式文件系统
    sudo mkfs -t ext3 /dev/sdb1 在此设备上创建一个ext3文件系统
  fsck
   

3. 创建一个CD-ROM的映像文件
   
   dd if=/dev/cdrom of=ubuntu.iso
